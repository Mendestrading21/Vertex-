# VERTEX CONSTITUTION

> Loi suprême du produit. **Courte. Impérative. Durable.** En cas de contradiction
> avec tout autre document, code ou commentaire, **la Constitution l'emporte.**
> Le code se conforme à la Constitution — jamais l'inverse.

## Les lois immuables

1. Vertex est un **instrument de décision**, pas un agrégateur de données.
2. Une page possède **une seule mission**.
3. Une section répond à **une seule question**.
4. Un graphique doit **améliorer une compréhension ou une décision** — sinon il disparaît.
5. **Réponse d'abord**, justification ensuite, expertise à la demande.
6. Une donnée possède **une seule source canonique** et **un seul domicile principal**.
7. Toute donnée affiche **sa source, son unité, sa période et sa fraîcheur**.
8. **Zéro, absent, indisponible, périmé, estimé et démo** sont des états différents.
9. Vertex préfère dire **« je ne sais pas »** plutôt que fabriquer une certitude.
10. Une couleur possède une **signification stable et financièrement cohérente**.
11. Le **calme, la hiérarchie et la lisibilité** sont des fonctionnalités.
12. L'utilisateur ne doit **jamais chercher ce qu'il doit regarder ensuite**.
13. La densité n'est autorisée que **lorsqu'elle est ordonnée**.
14. Les détails experts ne doivent **jamais écraser la décision principale**.
15. Une opportunité doit montrer le **risque, le scénario probable, le scénario
    exceptionnel, l'invalidation et le catalyseur**.
16. **Conviction = données + probabilités**, jamais émotion.
17. Vertex **protège d'abord le capital**.
18. Aucune interface ne doit encourager le **renforcement d'une position perdante**.
19. Les gagnants sont **réévalués selon la thèse**, jamais vendus automatiquement
    uniquement à cause d'un gain.
20. Toute **modification de calcul exige un test et une preuve**.
21. IBKR reste strictement **READONLY**, sans aucun chemin d'exécution d'ordre.
22. Les **données réelles, le mode démo, le fonctionnement sans IBKR et les états
    dégradés** doivent tous rester honnêtes.
23. Aucun nouveau composant, graphique, endpoint ou espace ne peut être ajouté
    **sans vérifier qu'un équivalent n'existe pas**.
24. Le **code mort est supprimé après preuve** ; l'historique Git suffit.
25. Aucun lot n'est terminé avec des **tests rouges, une erreur console, une
    régression mobile ou une contradiction non documentée**.
26. Vertex ne cherche pas à impressionner : il cherche à **rendre l'utilisateur
    meilleur investisseur**.

## Règle de priorité documentaire

En cas de conflit entre documents, l'autorité décroît dans cet ordre :

1. `VERTEX_CONSTITUTION.md` — **suprême**
2. `VERTEX_PRODUCT_BIBLE.md`
3. `PRODUCT_EXPERIENCE_REVIEW.md`
4. `SKILL.md` et `FRAMEWORK.md`
5. Documents d'audit et plans de PR
6. Commentaires historiques du code

> Aucune PR ne peut modifier la Constitution ou la Product Bible pour « faire
> passer » du code. Si le code contredit la Constitution, c'est le code qui a tort.
