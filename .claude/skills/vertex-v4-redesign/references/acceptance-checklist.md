# VERTEX — Checklist d’acceptation

## Global

- [ ] fond noir neutre, sans dominante bleue ou verte ;
- [ ] cartes en verre gris réellement transparentes ;
- [ ] navigation active neutre argentée ;
- [ ] vert uniquement positif ;
- [ ] rouge uniquement négatif/risque ;
- [ ] orange uniquement prudence/incertitude ;
- [ ] aucune donnée fictive ;
- [ ] aucune régression READONLY ;
- [ ] aucune erreur console ;
- [ ] aucun débordement horizontal ;
- [ ] focus clavier visible ;
- [ ] responsive laptop et tablette ;
- [ ] états loading, empty, error, stale, partial ;
- [ ] service worker bumpé ;
- [ ] tests ciblés et suite complète verts.

## Graphiques

- [ ] question métier visible ;
- [ ] unité et période visibles ;
- [ ] source et fraîcheur disponibles ;
- [ ] tooltip cohérent ;
- [ ] resize propre ;
- [ ] destruction/registry sans fuite ;
- [ ] axes honnêtes ;
- [ ] couleurs sémantiques ;
- [ ] benchmark gris ;
- [ ] fallback honnête ;
- [ ] pas de bleu par défaut.

## Page

- [ ] objectif compris en moins de cinq secondes ;
- [ ] première hauteur d’écran consacrée à la décision ;
- [ ] informations secondaires sous le fold ou dans une vue dédiée ;
- [ ] filtres utiles et persistants ;
- [ ] tables scannables ;
- [ ] actions non destructives ;
- [ ] données personnelles et signaux théoriques clairement séparés.
